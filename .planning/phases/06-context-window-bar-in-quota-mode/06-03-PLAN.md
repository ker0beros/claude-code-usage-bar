<objective>
Wire the `show_context` toggle (plan 06-01) and the per-style ctx renderers (plan 06-02) into the
live render path and the preview. In quota mode, when show_context is on, core drops the model's
`(used/size)` suffix and passes the flag to the renderer so the ctx bar is drawn; preview mirrors the
same gating so `cs preview` shows what the status line will show. This closes CTX-01 (bar appears in
real quota-mode renders) and CTX-03 (toggle actually controls it end-to-end; off = unchanged).

Purpose: 06-02 made the renderers capable; this plan makes them fire. Per the LOCKED "Suffix source"
decision (06-CONTEXT.md), the `(used/size)` suffix is appended in core.py (official-quota branch
~1383-1387 and waiting branch ~1469-1472). Core must stop appending it when show_context is on and
thread the flag into every render branch. preview.py builds the suffix independently and must gate it
the same way, and additionally compute ctx_pct (which it does not currently pass) so the bar renders.

Output: suffix gating + show_context passthrough across core's three render branches; preview threading
+ ctx_pct computation; a core-seam integration test and a preview test.
</objective>
