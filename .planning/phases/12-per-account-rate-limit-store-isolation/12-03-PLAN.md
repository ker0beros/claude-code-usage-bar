the legacy hardcoded resolver and the bug persists.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| session stdin → core.main → predict.account_id | `transcript_path` from stdin drives per-session resolution; must stay daemon-safe and never crash the render. |
| per-session env → resolution | `_effective_env` (per-session, core.py:1064-1065) is used instead of the daemon's frozen os.environ. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-12-05 | Tampering | core.main → account_id(transcript_path) | medium | mitigate | Resolution delegates to predict/account, which reuse Phase 11's `projects`-parent transcript guard; no new path is built from raw untrusted input in core. |
| T-12-06 | Denial of Service (self) | _resolved_account_uuid resolution | medium | mitigate | Wrapped in `try/except Exception: _resolved_account_uuid = None`; a resolution failure degrades to today's legacy keying, never an exception into the render fast path (mirrors the email-chip block at 1171-1180). |
| T-12-07 | Information Disclosure | daemon frozen os.environ leak | low | mitigate | Uses per-session `_effective_env`, not the daemon's start-time os.environ, so one session cannot key another session's account. |
| T-12-SC | Tampering | package installs | low | accept | No npm/pip/cargo installs — a call-site wiring change only (RESEARCH: Package Legitimacy Audit not applicable). No legitimacy gate required. |
